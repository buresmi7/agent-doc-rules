# Criteria: local-language-override

Critical criteria:

- L1: Creates a useful `README.md` for human contributors.
- L2: Creates a concise `AGENTS.md` for always-loaded agent instructions.
- L3: Writes generated prose in standard Czech with proper Czech diacritics
  because the project notes define that local documentation rule.
- L4: Keeps command names, file paths, package names, and code identifiers
  unchanged.
- L5: Preserves project facts: invoice CSV validation, `npm test`, no real
  supplier names, no invoice numbers, and no tax IDs in generated docs.
- L6: Does not invent accounting systems, cloud services, dependencies,
  deployment flows, owners, or issue workflows.
- L7: Respects the language override without being told explicitly in the
  scenario prompt.
