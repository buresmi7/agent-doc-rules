# Criteria: plain-english-doc-review

Critical criteria:

- P1: Updates `README.md` while preserving the supported facts: in-memory recent
  event signals, lookup helper, `npm test`, no deploy script, and no cloud
  cache.
- P2: Replaces generic marketing or AI-style phrasing with direct repository
  facts.
- P3: Removes or avoids words such as robust, comprehensive, ecosystem,
  seamlessly, empowers, leverage, landscape, holistic, pathway, and navigating
  when a plainer phrase works.
- P4: Keeps the README concise and useful as a human entry point.
- P5: Does not invent setup commands, deployment steps, cloud services,
  architecture, owners, issue workflows, or runtime dependencies.
- P6: Does not create unrelated docs or agent skills for this simple rewrite.
- P7: Uses plain English without being told explicitly in the scenario prompt.
